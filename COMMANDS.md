# Puntu — справочник команд и горячих клавиш

Полный список текущего функционала. Puntu теперь работает как **IBus-движок** (см. `README.md`);
старый uinput-демон остаётся в дереве как запасной вариант для не-GNOME и описан в самом конце.

## Движок IBus (`puntu-ibus`)

```sh
puntu-ibus <команда>
```

| Команда | Что делает |
|---|---|
| `puntu-ibus install` | записать компонент `~/.local/share/ibus/component/puntu.xml` (`install.sh` делает это сам + копирует в `/usr/share/...`) |
| `puntu-ibus enable` | сделать Puntu активным движком IBus |
| `puntu-ibus disable` | вернуться на обычную раскладку `xkb:us::eng` (переопределяется `PUNTU_FALLBACK_ENGINE`) |
| `puntu-ibus status` | активный движок + зарегистрирован ли Puntu + сколько процессов движка запущено |
| `puntu-ibus` (без аргументов) | запуск как движок — это вызывает сам `ibus-daemon` через поле `<exec>` |

> Переключаться на движок удобнее через **Super+Space** или значок источника ввода → «Puntu».
> Логи движка: `PUNTU_LOG=puntu=debug ibus restart`, затем `journalctl --user -f | grep puntu`.

## Режимы движка

| Режим | Как включить | Что делает |
|---|---|---|
| **EN auto-correct** (по умолчанию) | активен сразу после переключения на Puntu | печатаете по-английски; слова в неверной раскладке (`ghbdtn`) на разделителе конвертируются в русские (`привет`) |
| **RU direct** | **тап Ctrl** (нажать-отпустить без других клавиш) | каждая клавиша сразу даёт русскую букву — печать по-русски без отдельного источника `ru` |

Повторный тап Ctrl возвращает обратно; текущий режим показывается подсказкой
(`EN auto-correct` / `RU direct`).

## Горячие клавиши движка

По умолчанию (раздел `[ibus_hotkeys]` в `~/.config/puntu/config.toml`):

| Клавиша | Действие |
|---|---|
| **тап Ctrl** | переключить режим EN auto-correct ↔ RU direct (`mode_toggle`) |
| `Ctrl+` `` ` `` | перевернуть последнее слово между прочтениями — вернуть ровно то, что было набрано (`undo_key`); отмена авто‑конверсии добавляет слово в learned‑список, и оно больше не конвертируется |
| **тап Ctrl+Shift** | перевести **выделенный мышью** текст в другую раскладку (`convert_last`) |
| `Ctrl+Alt+S` | перевести **выделенный мышью** текст (обычная клавиша — если тапы не срабатывают) (`convert_selection_key`) |

> «Тапы» модификаторов срабатывают на **отпускании**; IBus доставляет release-события не на каждой
> конфигурации, поэтому для надёжности есть обычный хоткей `Ctrl+Alt+S`. Любую привязку можно
> переназначить в `config.toml`; `"none"` отключает.

## CLI (`puntu`)

Движок делит словари и конфиг с CLI, поэтому учат его теми же командами.

```sh
puntu [--config <ПУТЬ>] <команда>
```

| Команда | Что делает |
|---|---|
| `puntu stdin` | подать слова со stdin и напечатать решение детектора (без устройств/прав — для проверки) |
| `puntu build-dict <вход> [<выход>]` | собрать большой словарь RU в FST (по умолчанию `~/.config/puntu/russian.fst`) |
| `puntu config show` | показать текущий конфиг (TOML) |
| `puntu config set <ключ> <значение>` | изменить настройку и записать в `config.toml`: флаги (`dry_run`, `paste_convert`, `enable_modifier_taps`), числа (`tap_max_hold_ms`, `suggest_after` — 0 отключает предложение запомнить), хоткеи (`undo_key`, `convert_selection_key`, `remember_key`, `mode_toggle`, `convert_last`) |
| `puntu --version` / `puntu --help` | версия / помощь (работает и для подкоманд: `puntu dict --help`) |

### Словарь (`puntu dict …`)

| Команда | Что делает |
|---|---|
| `puntu dict list [--manual\|--learned\|--force\|--commands] [--ru\|--en]` | показать список (по умолчанию — исключения) |
| `puntu dict add <слово> [--ru\|--en]` | **исключение** — никогда не исправлять это слово |
| `puntu dict add <слово> --force [--ru\|--en]` | **всегда** конвертировать это слово |
| `puntu dict add <слово> --command` | добавить в список команд (защита ввода в терминале) |
| `puntu dict learn <слово> [--ru\|--en]` | **распознаваемое слово** (напр. название сервиса) — теперь его кривая раскладка переведётся: `ешлещл → tiktok` |
| `puntu dict ui` | **окно словаря** (zenity): пары «слово / как набирается», добавление и удаление |
| `puntu dict rm <слово>` | удалить слово из всех списков |
| `puntu dict forget <слово> [--ru\|--en]` | убрать **одно** авто‑обученное слово |
| `puntu dict clear-learned` | очистить все авто‑обученные слова |

> Язык определяется автоматически по слову (кириллица → ru, латиница → en); `--ru`/`--en` переопределяют.

### Инструмент модели

```sh
puntu-train <слово> [<слово> …]   # показать ru/en баллы слов (для подбора порогов)
```

## Поведение детекции

- авто‑конвертация неверной раскладки в обе стороны (RU↔EN), атомарно через `commit_text`;
- автозамена только в **доверенном контексте** (слово набрано подряд, курсор в конце); навигация
  и правка середины слова отключают замену, чтобы не портить текст;
- односимвольные слова: `ш→i`, `b→и`; валидные одиночные (`и`,`a`,`я`,`в`…) не трогаются;
- защита команд (`git`,`ls`,… + токены с `/`, `-флагами`, путями, цифрами);
- **точный матч по словарю**: формы слов (`кнопку`,`настройками`,`приложениями`) переводятся надёжно;
  имена сервисов добавляются через `dict learn`;
- **большой словарь RU** (~1.5 млн словоформ, fst ~2 МБ, ~13 МБ RAM) собирается из
  `dictionaries/russian.utf-8`: `puntu build-dict dictionaries/russian.utf-8` →
  `~/.config/puntu/russian.fst` (подхватывается при старте движка).

## Файлы конфигурации (`~/.config/puntu/`)

| Файл | Содержимое |
|---|---|
| `config.toml` | пороги детекции, раздел `[ibus_hotkeys]` (хоткеи движка), `dry_run`, `paste_convert` |
| `manual.{ru,en}.txt` | исключения (никогда не исправлять) |
| `force.{ru,en}.txt` | всегда конвертировать |
| `learned.{ru,en}.txt` | авто‑обученные (через отмену) |
| `words.{ru,en}.txt` | распознаваемые слова (`dict learn`) |
| `commands.txt` | пользовательские команды |

Все списки — текстовые (по слову на строку), правятся вручную.

## Установка

```sh
./install.sh                 # одна команда: зависимости + сборка + регистрация в IBus + активация
```

IBus-движку **не нужны** группа `input`, udev, uinput, systemd-сервис и релогин. Подробности —
в `README.md`. Пересобрать/переустановить бинарь вручную:

```sh
cargo install --path . --no-default-features --features ibus --force
```

## Переключение фронтендов (`puntu mode`)

Один переключатель между IBus-движком и запасным uinput-демоном; каждая команда оставляет
активным ровно один фронтенд (или ни одного). Не требует sudo (`systemctl --user` + `ibus engine`).

| Команда | Что делает |
|---|---|
| `puntu mode ibus` | остановить uinput-демон и активировать IBus-движок (рекомендуется; сначала `puntu-ibus install`) |
| `puntu mode uinput` | деактивировать IBus-движок и запустить uinput-демон |
| `puntu mode off` | выключить оба фронтенда |
| `puntu mode status` | показать, какой фронтенд активен (движок IBus + жив ли демон) |

> Резервный движок при выходе из `ibus`-режима — `xkb:us::eng`; переопределяется переменной
> `PUNTU_FALLBACK_ENGINE`.

## Запасной вариант: uinput-демон (не-GNOME)

Старый путь без IBus (читает клавиатуру через evdev, исправляет через буфер обмена). Собирается
фичей `daemon` (по умолчанию) и управляется через systemd; требует группу `input`, `/dev/uinput`
и `wl-clipboard`. На GNOME/Wayland используйте IBus-движок — он точнее и без гонок.

```sh
puntu run [--dry-run]                    # запустить демон (детект + исправление)
puntu pause | resume | status            # управление работающим демоном (только uinput-демон;
                                         #   IBus-движок сокета не имеет — см. `puntu mode`)
systemctl --user enable --now puntu      # автозапуск (после релогина в группе input)
sg input -c "PUNTU_LOG=debug puntu run"  # запуск с группой input без релогина, с логом
```
